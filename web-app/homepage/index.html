Visão técnica

O IAprovale é um sistema real em produção: backend, cliente desktop e integrações construídos com foco em confiabilidade, segurança e simplicidade de manutenção.

Arquitetura

Backend em FastAPI (Python), hospedado no Render.com. Banco de dados Postgres gerenciado via Supabase, com persistência real de usuários, perfis de formação e histórico de concursos vistos. Cliente desktop em Python com Tkinter + ttkbootstrap (temas claro/escuro), executando chamadas de rede em threads separadas para nunca travar a interface.

Inteligência artificial

A busca em linguagem natural é interpretada pela API da DeepSeek, que traduz a frase do usuário em parâmetros estruturados de busca — a pessoa escreve como fala, o sistema consulta como máquina.

Fonte de dados

Um servidor MCP (Model Context Protocol) oficial da PCI Concursos é consultado com os parâmetros estruturados — sem scraping, dados sempre atualizados e estruturados na origem.

Filtragem e deduplicação

Os resultados são filtrados por status de inscrição aberta e compatibilidade de formação. Buscas repetidas do mesmo usuário destacam apenas concursos novos, com deduplicação por ID.

Segurança

Acesso somente por convite — sem autocadastro público; só o administrador cria contas. Senhas nunca ficam em texto puro (hash Argon2id). Autenticação via token JWT. Chaves de API sensíveis (DeepSeek) nunca são expostas ao cliente — toda chamada de IA passa pelo backend.

Qualidade

Desenvolvimento orientado a testes automatizados (TDD / pytest) tanto no backend quanto no cliente desktop — cada funcionalidade nasce com seus testes.
